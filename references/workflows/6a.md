## Stage ownership in the imperial model
在朝廷式模型中的阶段归属

- Align: `taizi` + `zhongshu`
- Align（目标对齐）：`taizi` + `zhongshu`
- Architect: `zhongshu`
- Architect（架构设计）：`zhongshu`
- Atomize: `zhongshu`
- Atomize（原子化拆解）：`zhongshu`
- Approve: `menxia`
- Approve（审批）：`menxia`
- Automate: `shangshu` dispatching `bingbu`, `libu`, `gongbu`, `xingbu` as needed
- Automate（执行）：由 `shangshu` 根据需要调度 `bingbu`、`libu`、`gongbu`、`xingbu`
- Assess: `shangshu` + `libu`
- Assess（评估）：`shangshu` + `libu`

## Control rules
控制规则

- do not skip stages
- 不允许跳过阶段
- do not code before Align and Architect are complete
- 在 Align 和 Architect 完成之前，不要进入编码
- if approval fails, return to Architect or Atomize
- 如果审批失败，回退到 Architect 或 Atomize
- each atomic task should map to one executable change unit where feasible
- 在可行情况下，每个原子任务应对应一个可独立执行的变更单元
